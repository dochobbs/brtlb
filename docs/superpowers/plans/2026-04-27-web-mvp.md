# brtlb ‚Äî Web MVP Plan (Phase 4-MVP)

**Goal:** Working web app today. Pediatrician opens brtlb in a browser, pastes their AssemblyAI + LLM keys, records a visit, gets a SOAP note, copies it into their EHR. Browser only (desktop primary, mobile-responsive secondary). PHI flows directly client ‚Üí AssemblyAI ‚Üí LLM provider. Nothing on a brtlb-controlled server.

**Stack reuse:** `@brtlb/pipeline` (Anthropic + OpenAI adapters work in browser), `@brtlb/prompts` (SOAP + narrative), `@brtlb/ui` (Lockup, Button), `@brtlb/db` is **NOT** used (Node-only); IndexedDB via `idb` instead.

**Deferred from full plan:** Vertex provider (server-side auth needed), encryption-at-rest (BYO-keys means user is the security boundary), biometric lock, PDF/DOCX export, native share, multi-template UI, age-aware Well-Child variants, ICD-10 picker, regenerate-with-roles flow.

**v0 ship criteria:** record ‚Üí transcribe with diarization ‚Üí SOAP note ‚Üí manual edit ‚Üí copy markdown to clipboard. Nothing else.

## Tasks

### T1 ‚Äî Browser-safe AssemblyAI client variant

`@brtlb/pipeline` currently reads audio via `node:fs/promises`. Add a sibling export `transcribeBlobWithAssemblyAi` that takes a `Blob`/`ArrayBuffer` instead of a path. Refactor internals: extract `uploadAudioBody`, `requestTranscript`, `pollTranscript` so both wrappers reuse them.

Tests: 2 new (Blob upload happy path, Blob error path) using mocked fetch.

### T2 ‚Äî `apps/web-mvp` scaffold

Mirror `apps/web` structure: Vite + React 19 + Tailwind v3 + TS. Different port (5181 to avoid collision). Brand-applied home screen as starting point.

Manual verification: `pnpm --filter @brtlb/web-mvp dev` serves at localhost:5181.

### T3 ‚Äî IndexedDB persistence + Zustand store

`lib/db.ts`: `idb`-backed wrapper exposing `recordings.put(rec)`, `recordings.list()`, `recordings.get(id)`, `recordings.delete(id)`, `audio.put(id, blob)`, `audio.get(id)`. Two object stores: `recordings` (metadata) and `audio` (blobs).

`store.ts`: Zustand store holding `settings` (loaded from localStorage on init), `view` ('home' | 'settings' | 'record' | 'review'), `currentRecordingId`. Provides `saveSettings()`, `setView()`, `selectRecording(id)`.

Tests: db wrapper round-trip, store actions.

### T4 ‚Äî Settings screen

Form with provider picker (Anthropic / OpenAI), API key field (password input), model selection (preset list per provider, with an "other" text input), AssemblyAI key field. Save button persists to localStorage. Test button does a 1-message round-trip to verify keys work.

Pre-flight pediatric defaults: Anthropic ‚Üí `claude-sonnet-4-6`, OpenAI ‚Üí `gpt-4o`.

### T5 ‚Äî Recorder component + Record screen

`Recorder` hook: encapsulates `navigator.mediaDevices.getUserMedia`, `MediaRecorder` with `audio/webm;codecs=opus`, real-time level metering via Web Audio analyzer node, pause/resume/stop. Returns `{ start, stop, pause, resume, levels[], elapsedMs, state }`.

`Record` screen: mode toggle (ambient/dictation), big record button, live level bars, pause/stop. On stop, persist Blob + metadata to IndexedDB, transition to processing ‚Üí Review.

### T6 ‚Äî Pipeline runner with progress UI

`lib/pipeline-browser.ts`: takes the Blob + provider config + AssemblyAI config, calls `transcribeBlobWithAssemblyAi`, then constructs the right adapter (Anthropic/OpenAI), calls `composeNotePrompt` + adapter's `generateNote`, returns `{ transcript, note }`.

UI: progress states (uploading ‚Üí transcribing ‚Üí generating) shown as a stepper.

### T7 ‚Äî Review screen

Two-pane (or stacked on mobile): left = diarized transcript with speaker chips; right = generated note in a textarea (editable). Copy-as-Markdown and Download buttons. Status bar: "Saved locally" + delete button.

### T8 ‚Äî Home screen + recording list + final polish

Home screen: brand lockup at top, big "New recording" button, then list of past recordings (date, duration, mode, status, "Open" button). Empty state if none. Settings link in header.

Manual smoke test: full flow with real keys (user-provided in dev). Build for production. Confirm size is reasonable (<500KB gzipped target).

## Out of scope for v0 (parking lot)

- Vertex AI (skip; harder browser auth path)
- PDF/DOCX export (markdown copy + plain-text download only)
- Multi-template / pattern picker (SOAP + narrative hardcoded)
- Speaker rename + regenerate flow
- Auto-purge audio
- PWA install banner
- Service worker / offline support
- Mobile-Safari recording quirks (test path only on desktop Chrome/Safari/Firefox for v0)
- Background recording (browser tab must stay foreground)
- Encrypted settings (keys in plain localStorage; warn user)
- Real-key smoke tests in CI
