Cross-platform behavior summary

Platform Wake Lock Lock kills mic Vibration alert Gap warning
iOS Safari (16.4+) ✅ Works in tab ❌ Within seconds ❌ No API on iOS ✅ Shows on return
iOS Safari PWA (17.4+) ✅ Works ❌ Within seconds ❌ No API ✅ Shows on return
Android Chrome (84+) ✅ Works 🟡 5s–5min grace ✅ Phone buzzes ✅ Shows on return
Desktop browsers ✅ Display sleep prevented ❌ on system sleep / 🟡 on display sleep ❌ N/A ✅ Shows on return

The honest takeaway: the only platform-independent guarantee is the gap warning that surfaces on return. Wake Lock prevents accidental auto-lock everywhere. Vibration only helps Android users notice the lock. Manual screen-lock during recording is unsupported in browser environments — only a native shell can keep the mic alive past lock on iOS.
