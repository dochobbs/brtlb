Why this design (click for the rationale)
  • The active tab IS the scope. One mental model. No second "edit which patient?" dropdown to remember. If you're looking at Tommy, you're working on Tommy.
  • "All combined" is opt-in for global edits. Want to tweak all three at once ("add return precautions to all plans")? Switch to that tab — the Tweak prompt switches to multi-patient mode and edits across patients while preserving each kid's section.
  • Section paste, Tweak, Regenerate, Copy, Share, Email, Download — all retarget when you switch tabs. No surprises about which patient an action applied to.
  • Templates are per-patient. Each tab shows the template that was auto-detected for that kid (Sick Visit for Tommy, Well Child for Lily and Max). Changing the dropdown on Tommy's tab and clicking Regenerate only affects Tommy.
  • Single-patient visits look exactly like today. The tabs only appear when there's more than one patient. Solo recordings keep the streamlined single-pane UI.
  • No data model change. Notes are still stored as one concatenated markdown string. Tabs are derived at render time by parsing the H2 headers. When you tweak Tommy's section, we splice the new content back in place. Lily and Max's sections stay untouched in the underlying string.
